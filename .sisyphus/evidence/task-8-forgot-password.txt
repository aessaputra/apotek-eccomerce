# Task 8 Forgot Password request scene evidence

## Files changed
- `scenes/auth/ForgotPassword.tsx` now renders the real request screen instead of the Task 6 null placeholder.
- `__tests__/scenes/ForgotPassword.test.tsx` was created with centralized scene coverage via `@/test-utils/renderWithTheme`.
- `scenes/auth/index.ts` did not need changes because it already exported `ForgotPassword`.

## Implementation assertions
- Screen uses the existing auth visual structure: `SafeAreaView`, `KeyboardAvoidingView`, `ScrollView`, logo/header/form-card layout, Tamagui `YStack`/`XStack`/`Text` tokens, `getCardShadow`, and `PRIMARY_BUTTON_TITLE_STYLE`.
- Screen has exactly one `EmailInput`, one primary submit button, and a Login CTA routed to `/(auth)/login`.
- Valid submissions call `requestPasswordReset(trimmedEmail)` through `@/services/auth.service`; there is no direct Supabase import/call in the scene.
- Empty/invalid email is handled client-side with Indonesian validation copy and does not call `requestPasswordReset`.
- Successful reset requests show `AUTH_FORGOT_PASSWORD_GENERIC_SUCCESS_MESSAGE`.
- Retryable/rate-limited errors show actionable localized copy via `getAuthErrorMessage(error)` and are not converted to generic success.
