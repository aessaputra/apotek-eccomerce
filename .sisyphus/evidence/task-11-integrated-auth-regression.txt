# Task 11 integrated auth regression and recovery hardening evidence

## Coverage assessment before changes

No new auth tests or production fixes were required for Task 11. Existing Tasks 1-10 coverage already satisfies the integrated behavior checklist:

- Login -> Forgot Password navigation: `__tests__/scenes/Login.test.tsx` asserts pressing `Lupa Password?` calls `router.push('/(auth)/forgot-password')`.
- Forgot Password valid email success/check-email copy: `__tests__/scenes/ForgotPassword.test.tsx` asserts a trimmed valid email calls `requestPasswordReset(...)` and renders `AUTH_FORGOT_PASSWORD_GENERIC_SUCCESS_MESSAGE`.
- Recovery link cold start to Reset Password: `__tests__/scenes/ResetPassword.test.tsx` asserts route params `{ token_hash, type: 'recovery' }` call `verifyEmailOtp({ tokenHash, type: 'recovery' })` before rendering the form.
- Reset Password success flow: `__tests__/scenes/ResetPassword.test.tsx` asserts `updatePassword('password1')` runs before `signOut()`, then routes to `/(auth)/login` with `{ resetSuccess: LOGIN_RESET_SUCCESS_MESSAGE }`.
- Normal Login/Register coverage: `__tests__/scenes/Login.test.tsx` and `__tests__/scenes/SignUp.test.tsx` cover required-field, invalid email, service errors, verify-email redirects, and session/non-session register behavior.
- Google OAuth cancellation/error coverage: `__tests__/scenes/Login.test.tsx` asserts cancellation is ignored without surfacing the cancellation message and non-cancellation OAuth errors render mapped service copy.
- AuthProvider push-token/session lifecycle coverage: `__tests__/app/auth-provider.lifecycle.test.tsx` covers `INITIAL_SESSION`, `SIGNED_IN`, `TOKEN_REFRESHED`, `SIGNED_OUT`, and stale sign-in callback behavior.
- Recovery-aware root guard coverage: `__tests__/app/root-layout.notifications.test.tsx` asserts logged-in users on `['(auth)', 'reset-password']` are not navigated to `/home`.
- Service redirect/helper coverage: `__tests__/services/auth.service.test.ts` asserts `createPasswordRecoveryRedirectUri()` uses `{ scheme: 'apotek-ecommerce', path: 'reset-password' }` and returns `apotek-ecommerce://reset-password` in the mocked native redirect environment.
